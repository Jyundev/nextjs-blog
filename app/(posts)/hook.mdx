기존 방식 : 상태들을 useState 로 관리하는 제어 컴포넌트

--
useForm 요약 (한국어)
useForm은 React Hook Form에서 폼 상태를 관리하는 가장 핵심적인 훅이야.
폼 입력값, 유효성 검사, 에러 상태, 제출 처리 같은 걸 쉽게 관리할 수 있도록 도와줘.

🛠️ useForm 주요 옵션들

옵션 설명 기본값
mode 유효성 검사를 언제 할지 설정 (onSubmit, onChange, onBlur 등) onSubmit
reValidateMode 이미 유효성 검사를 한 필드를 다시 검사할 타이밍 (onChange, onBlur) onChange
defaultValues input의 초기 값 설정 {}
resolver 외부 라이브러리(yup, zod 등)로 유효성 검사 연결할 때 사용 없음
criteriaMode 여러 에러를 한번에 모을지 (firstError, all) firstError
shouldFocusError 에러가 발생한 첫 번째 필드로 포커스 이동 여부 true
📦 useForm이 리턴해주는 것들

리턴 값 설명
register input에 연결하는 함수
handleSubmit 제출 이벤트 핸들러
watch 특정 필드 값 또는 전체 폼 값 관찰
setValue 특정 필드 값을 직접 설정
getValues 현재 폼 값을 가져옴
reset 폼 전체를 초기화
trigger 특정 필드 유효성 검사 실행
formState 에러, dirty, touched, isSubmitting 같은 상태들
🧠 예를 들어
mode: "onChange" 를 주면, 입력할 때마다 바로 유효성 검사를 해!

defaultValues 를 쓰면, 서버에서 받아온 초기 데이터를 폼에 미리 채워넣을 수 있어.

요약 정리
useForm은 폼 상태를 쉽게 관리하게 해줌.

register, handleSubmit, watch, formState 가 핵심.

옵션 (mode, defaultValues, resolver 등) 을 설정하면 더 세밀하게 커스터마이징 가능.

useForm 하나로 폼 입력값, 유효성 검사, 에러 관리를 모두 처리할 수 있어!

항목 getValues watch
실시간 반영 ❌ 실시간 반영되지 않음. 호출 시점의 값만 반환. ✅ 실시간으로 값의 변화를 감지하여 반영.
리렌더링 ❌ 리렌더링을 유발하지 않음. ✅ 감지하는 값이 변경되면 리렌더링을 유발.
구독(Subscription) ❌ 값의 변화를 구독하지 않음. ✅ 특정 필드나 전체 폼의 값을 구독하여 변화 감지.
사용 목적 특정 시점의 폼 데이터를 가져올 때 사용. 예: 제출 시점의 데이터 확인. 입력값의 변화를 실시간으로 감지하여 UI에 반영할 때 사용.
성능 고려 ✅ 리렌더링이 없으므로 성능에 영향 없음. ❌ 값의 변화에 따라 리렌더링이 발생하므로 성능에 주의 필요.
