## 컨텍스트 스위칭

CPU가 여러 프로세스/스레드를 번갈아가며 실행할 때, 현재 작업 상태를 저장하고 다음 작업의 상태를 불러오는 과정

멀티태스킹이 가능하게 해주는 핵심 기술

### 중요성

CPU는 한 순간에 하나의 명령어만 실행 가능

하지만 동시에 여러 작업을 하는 것처럼 보여야 함

그래서 운영체제가 일정 시간마다 작업을 교체 → 시분할 시스템

### 과정

컨텍스트(Context) = 실행에 필요한 정보 (레지스터, 프로그램 카운터, 스택 포인터 등)

현재 실행 중인 프로세스/스레드의 상태 저장 (PCB: Process Control Block)

새로 실행할 프로세스/스레드의 상태 불러오기

CPU 제어권을 넘김

### 오버헤드

Context Switching 비용

- 상태 저장 & 복원하는 데 시간이 듦

- 캐시 메모리(캐시 히트율)도 초기화되어 성능 저하

따라서 스위칭이 너무 자주 일어나면 오히려 비효율적

## 예시

웹 브라우저에서 유튜브 시청 + 카톡 웹 사용 + 구글 드라이브 문서 작성 → CPU가 컨텍스트 스위칭을 하면서 각각 돌아가게 만듦

Node.js의 이벤트 루프 → 비슷한 개념으로, 비동기 처리할 때 태스크 간 전환이 일어남

### 키 포인트

- 멀티코어 CPU에서는 스위칭보다 코어 간 병렬 실행이 더 효율적

- 실무에서는 스위칭 오버헤드를 줄이기 위해:

  - 비동기/이벤트 기반 처리 (Node.js, 브라우저 이벤트 루프)

  - 멀티스레드 대신 워커 풀(Thread Pool) 사용
